lofi.css v0.1

A tiny hand-drawn wireframe framework in a single CSS file. Paper-scribble look in the spirit of Excalidraw and tldraw — wobbly borders, marker highlights, squiggly underlines — for lo-fi mockups, prototypes, and docs where you want people to discuss structure and flow, not visual polish.

Grayscale only. No JavaScript. No dependencies. Bare HTML elements are styled automatically — classes are optional extras. Pixel-perfect Deliberately imperfect.

See it in action: example mockup · icon gallery

Get started

  1. Download lofi.css and place it next to your HTML file.
  2. Add the stylesheet to your <head>, and the lofi class to <body> — the handwriting fonts are embedded in the CSS, so nothing is loaded from anywhere else:
<link rel="stylesheet" href="lofi.css">

<body class="lofi"> ... </body>
  1. Write plain semantic HTML. Headings, lists, tables, quotes, code, forms, details, dialog — everything below gets the sketchy treatment with zero classes. Use the .lf-* classes for components (cards, buttons, sticky notes, annotations).
Using Claude? Install lofi-mockups.skill (bundles the CSS) or read the plain SKILL.md. The skill enforces the house rules: strictly HTML + this CSS + text, grayscale only, and no emojis.

Icons

lofi.css ships 19 built-in scribble icons as an inline SVG sprite — no icon font, no external files. Paste the sprite once, then reference by id. Icons inherit text color and size with the text:

Need more than these? The full icon set has all 1745 Lucide icons redrawn in the same scribble style — searchable, click-to-copy, or download the whole sprite as lofi-icons.svg.

    Coverage notes

    Intentionally unstyled: non-rendering elements (head, meta, script, template, slot, datalist popup, track/source, map/area), pure wrappers (div, span, main, section, article, aside, header, footer, picture), and line-break controls (br, wbr). Obsolete elements (marquee, center, …) are out of scope.
